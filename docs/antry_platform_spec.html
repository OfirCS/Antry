Antry — Platform Spec · March 2026
What to build
MVP feature set, phased rollout, and recommended tech stack

Philosophy

Simple, visual, shareable. Every feature should be easy to market on social media. Each project a builder posts should be sharable as a standalone link with a beautiful preview image. The platform should feel like a gallery, not a dashboard.

Phase 1 — MVP (Months 1–3)

Start with the minimum needed to collect waitlist signups and run the first hackathon.

Must have

Waitlist page

Email capture, counter showing signups, clean landing with value prop. Deploy in Week 1.

Must have

Builder profile

Name, bio, skills, social links, and a gallery of project cards. Each profile has its own URL.

Must have

Project cards

Title, description, live demo link, video embed, tech stack tags, build time. Auto-generated OG image for sharing.

Must have

Auth

GitHub + Google login. Invite code system for early access control. Simple onboarding flow.

Must have

Hackathon page

Event theme, rules, timeline, prizes. Submission flow with demo link + description. Basic leaderboard.

Must have

Discovery feed

Browse projects by category (AI agents, web apps, tools). Trending + newest filters. Search by tags.

Phase 2 — Community (Months 4–6)

Phase 2

Social features

Like and save projects. Follow builders. Activity feed. Comment threads on project cards.

Phase 2

Sponsor dashboard

Companies can create and manage hackathon events, set prizes, review submissions, and access builder profiles.

Phase 2

Featured profiles

Premium tier for builders — featured placement in discovery, analytics on profile views, priority at events.

Phase 2

Builder analytics

Profile views, project views, demo clicks. Helps builders understand their reach and improve showcasing.

Phase 3 — Studio (Month 7+)

Phase 3

Studio applications

Builders can submit ideas for studio validation. Application form, review queue, status tracking.

Phase 3

Cohort pages

Public pages for each studio cohort. Timeline of their build. Updates and milestones. Good for PR.

Design principles

Demo-first, not CV-first Shareable on social media Beautiful OG previews Mobile-responsive Fast page loads Dark mode support Invite-only early access

MVP build estimate

A solo developer or a small team of two can ship the Phase 1 MVP in 3–4 weeks. The waitlist page alone can go live in 1–2 days. Prioritize shipping the landing page and collecting social proof before building the full product.

Week 1 priority

Ship the landing page. Register antry.co. Set up email collection. Post to X / LinkedIn / relevant Discord communities. That's the entire Week 1 goal. Nothing else matters until you have 100 people on a waitlist.
